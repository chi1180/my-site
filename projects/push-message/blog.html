Push 通知の実装方法

始めに

今回は、Push 通知を実装してみました。
ついこの前、前回紹介させていただいた課題管理アプリの ver.2 を作成しました。その際に、Push 通知を実装したのですが、なにかと苦戦したり、問題が発生したりとしたので、ここに記載しとこーかなと思います。
作成した課題管理アプリ（ver.2）については、また今度、書かせていただきたいと思います。

機能

普通の Push 通知です。
思った以上に簡単に実装できました。
でも、ちょっとしたところで悪戦苦闘したり、エラーが吐き出されてきたりと、いろいろあったです。

それでは以下に、どのようにして Push 通知を実装したもうたのか、記述させていただきます。

1. push.js を使用して、Push 通知の実装を行いました。（下記ページを参考にしました：参考ページ）
2. 実装は想像以上に単純で、簡単にコードを記述して実装できました。
3. エラーになりました。
4. 原因を探ると、どうやら script タグを記載する場所に問題があったようです。
以下のような script タグを記述する必要があるのですが、自分で記述する script ファイルを src とした script タグのように、body タグの中に記述するのではなく、head タグの中に記述する必要があり、そうすることでエラーもなくなり、通知もうまく動きました。
<script src="https://cdnjs.cloudflare.com/ajax/libs/push.js/1.0.12/push.min.js"></script>

しかし！
いざリリースしてみると、通知が実行されません！原因を探るため調べてみると、どうやら http 問題だったそうです。
Push 通知の実装は、セキュリティ上の理由から、https での通信でのみしか実行されないようです。
つまり、私が初めに使用していたサーバ（http）ではPush 通知の使用は不可！！となりますことが判明したので、課題管理アプリを GitHub にアップロードして、GitHub Pages を使用して公開するようにしました。
無料で https のサイトを持つことができるなんて…最高ですよね！

以上のようにしたことで、Push 通知もきちんと動作するようになり、オールオッケーなお話になりました！

追記：開発中は、Live Server という拡張機能を使用しながら、アプリを localhost で表示して確認していたのですが、どうやら localhost では、例外的に Push 通知を実行することが可能ということで、非常に学びになりました。
